# Renderスリープ対策

## やったこと

- `GET /health` エンドポイントを追加（`HealthController#show` が `{ status: "ok" }` を返す）
- cron-job.org（無料）から9時〜翌2時の間だけ定期pingを設定することでスリープを防ぐ

## 技術的判断の理由

**なぜ外部pingサービスを使うのか**
Renderの無料プランは15分間アクセスがないとスリープする。スリープ中はRails自体が止まるため、ローディング画面も含めてアプリ側では何も対処できない。外部から定期的にHTTPリクエストを送ってスリープを防ぐのが唯一の現実的な手段。

**なぜ時間帯を2時〜9時は対策なしにしたのか**
Renderの無料枠は月750インスタンス時間。24時間起こし続けると約744時間を消費しほぼ上限に達する。深夜〜朝はアクセスが見込めないため17時間/日に絞ることで月約527時間に抑え、220時間以上の余裕を確保した。

**なぜRender内のCron Jobではなくcron-job.orgなのか**
Renderの無料Cron Jobサービス自体もスリープ対象になりうる。外部サービスの方が信頼性が高い。
