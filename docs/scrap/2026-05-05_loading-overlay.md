# API通信中のローディングオーバーレイ

## やったこと

- Stimulus コントローラ `loading_controller.js` を新規作成
- フォーム送信時に薬剤師アイコンがぷかぷか浮くオーバーレイを全画面表示
- 結果ページへ遷移したタイミングで自然消滅する仕組み
- 服薬指導箋フォームは「指導箋を作成中…」、副作用レポートフォームは「副作用レポートを作成中…」とメッセージを出し分け

## 技術的判断の理由

**なぜ `data-turbo="false"` にしたのか**

当初はTurboが提供する `turbo:render` イベントでオーバーレイを消す案を検討した。しかしTurbo固有のライフサイクルイベントへの依存が増えると挙動が読みにくくなるため、フォームのみTurboを無効にして通常のHTTPリクエスト（POST → redirect → 新ページ読み込み）に切り替えた。新ページにはオーバーレイのDOMが存在しないので、何もしなくても自然に消える。ローディング表示中はユーザーに画面が見えていないため、TurboによるSPA的な遷移の恩恵（白フラッシュ防止など）は不要と判断した。

**なぜ薬剤師アイコンをアニメーションさせたのか**

スピナーや3点ドットも検討したが、このアプリの世界観（深夜のバーで柴犬薬剤師がカウンター越しに話しかける）に合わせ、薬剤師がふわっと浮かぶアニメーションを選んだ。待ち時間をコンテンツとして体験させるための演出。

**なぜアイコンのURLを `loading_icon_value` としてERBから渡すのか**

Railsのアセットパイプラインは本番環境でファイル名にフィンガープリント（ハッシュ）を付加する（例: `pharmacist-a1b2c3.png`）。JSから直接 `/assets/pharmacist.png` を書くと本番で404になる。`asset_path` ヘルパーを使ってERB側で解決したURLをdata属性に渡すことでこの問題を回避した。
