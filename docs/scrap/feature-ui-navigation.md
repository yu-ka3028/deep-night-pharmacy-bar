# UI ナビゲーション改善

## やったこと

- ランディングページを新設（`/`）し、機能1・機能2を2択カードで提示
- 機能1（服薬指導箋）を `/play` に移動
- 全ページ共通ナビ（トップ・服薬指導箋・副作用レポート）を追加
- 各結果ページにもう一方の機能への誘導リンクを追加
- バリデーションエラー後のリダイレクト先を `root_path` → `play_path` に修正

## ルーティング

| パス | アクション | 用途 |
|------|-----------|------|
| `/` | `pages#top` | ランディングページ（新設） |
| `/play` | `topics#show` | 機能1: 服薬指導箋 |
| `/side_effects/new` | `side_effects#new` | 機能2: 副作用レポート |

## 学び

- ランディングページは専用の `PagesController` に分けると責務が明確になる
- 共通ナビは `application.html.erb` に入れるだけでよい
- 結果ページから別機能への誘導リンクを置くと回遊率が上がる
