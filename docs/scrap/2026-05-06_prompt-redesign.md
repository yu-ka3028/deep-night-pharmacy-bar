# プロンプトリデザイン・Xシェア設計の学び

## 薬剤師の役割はサービスの世界観の根幹

「薬剤師は処方しない、処方は医師の仕事」という前提をプロンプトに明記しないと、
AIが自然に「処方します」と言ってしまう。
キャラクターの職業的リアリティはプロンプトで厳守ルールとして宣言する必要がある。

この世界の設定：
- ユーザーの回答・今日の活動 = 持ち込まれた薬
- 柴犬薬剤師 = それを確認・分析・指導する立場

## Xシェア文の設計原則

- **機能1（服薬指導箋）**：お題名はプライベートではないので使える。薬剤名＋副作用名＋副作用内容1文目でサービスのユーモアを伝える
- **機能2（副作用レポート）**：今日の活動はプライベートなので出さない。薬剤名＋副作用でサービスの特徴だけ伝える

共通原則：「サービスを知らない人が『なにこれ？』と思えるか」を基準に要素を選ぶ。

## 副作用の2段構成でツイートと詳細表示を両立

```
副作用名：（20文字以内・キャッチーな症状名）← ツイートに使う
副作用内容：（2〜3文・1文目は40文字以内）  ← 1文目はツイート、全文はアプリ
```

ビュー側で `split("。").first` で1文目を抽出するだけで棲み分けできる。
プロンプトのフォーマットとビューのパース処理は密結合なので、設計時に一緒に考える。

## 薬剤師コメントのレポート外分離

「今夜の処方」という項目名が薬剤師の役割と矛盾していた。
→ `---` セパレータでレポートと薬剤師コメントを分離し、コメントは吹き出しに表示。
公式文書（レポート）と会話（指導）を視覚的にも分けることで世界観が一貫する。

## キャラクター口調の注意点

「〜わん」などの動物語尾はわざとらしくなるため禁止。
薬剤師らしく丁寧でフランクな口調（「〜ですね」「〜ですよ」程度）の方がキャラクターとして自然に立つ。
